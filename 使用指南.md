## 项目概述

这是一个基于 **OpenWrt/ImmortalWrt** 的固件定制项目，使用 GitHub Actions 自动化编译固件。目前支持的设备包括：
- GL-inet AXT-1800
- TR-3000
- JDC-AX6600
- GL-MT5000
- GL-MT3600BE

项目结构：
```
/workspace/
├── .github/workflows/    # GitHub Actions 工作流配置
├── config/               # 设备配置文件
├── default-settings-m0eak/  # 默认设置包
├── files/                # 自定义文件（会复制到固件中）
├── sh/                   # 构建脚本
│   ├── scripts-part1.sh  # 第一部分（更新 feeds 前执行）
│   └── scripts-part2.sh  # 第二部分（更新 feeds 后执行）
└── README.md
```

## 编译固件的基本流程

### 方式一：使用 GitHub Actions 编译（推荐）

1. **Fork 仓库**：将 [fanchmwrt](https://github.com/fanchmwrt/fanchmwrt) 仓库 Fork 到你的 GitHub 账户下

2. **配置工作流**：在 `.github/workflows/` 目录下找到你需要的工作流文件，或创建新的。

3. **手动触发构建**：
   - 进入你的仓库
   - 点击 "Actions" 标签
   - 选择对应的工作流（如 "Build filogic"）
   - 点击 "Run workflow" 按钮
   - 选择分支并点击绿色的 "Run workflow" 按钮

4. **下载固件**：构建完成后，在 Release 页面或 Actions 产物中下载编译好的固件。

### 方式二：本地编译

1. **准备编译环境**（Ubuntu/Debian）：
```bash
sudo bash -c 'bash &lt;(curl -s https://build-scripts.immortalwrt.org/init_build_environment.sh)'
```

2. **克隆固件源代码**：
```bash
# 根据你需要的设备选择对应的分支
git clone -b mt5000 https://github.com/m0eak/fanchmwrt.git openwrt
cd openwrt
```

3. **更新和安装 feeds**：
```bash
./scripts/feeds update -a
./scripts/feeds install -a
```

4. **配置固件**：
```bash
# 使用已有配置（如 mt5000.config）
cp /path/to/mt5000.config .config
make defconfig

# 或使用菜单配置
make menuconfig
```

5. **编译固件**：
```bash
make download -j$(nproc)
make -j$(nproc)
# 编译出错时使用单线程查看详细错误
make -j1 V=s
```

6. **固件位置**：编译好的固件在 `bin/targets/&lt;target&gt;/&lt;subtarget&gt;/` 目录下

## 如何添加自己需要的插件

### 方法 1：修改配置文件（最简单）

1. 打开 [config/mt5000.config](file:///workspace/config/mt5000.config)（或你设备对应的配置文件）

2. 添加需要的软件包配置：
```bash
# 添加 luci-app-xxx 插件
CONFIG_PACKAGE_luci-app-xxx=y

# 添加依赖库
CONFIG_PACKAGE_xxx=y

# 添加中文语言包（如果有）
CONFIG_PACKAGE_luci-i18n-xxx-zh-cn=y
```

3. 示例：添加 AdGuard Home
```bash
CONFIG_PACKAGE_luci-app-adguardhome=y
CONFIG_PACKAGE_luci-i18n-adguardhome-zh-cn=y
```

### 方法 2：通过 scripts-part2.sh 添加第三方插件仓库

编辑 [sh/scripts-part2.sh](file:///workspace/sh/scripts-part2.sh)，在 `REPOS` 数组中添加你需要的仓库：

```bash
declare -A REPOS=(
    ["https://github.com/你的/插件仓库.git"]=""  # 空字符串表示默认分支
    ["https://github.com/你的/另一个仓库.git"]="分支名"  # 指定分支
    # 现有仓库保持不变...
)
```

脚本会自动：
1. 克隆这些仓库到 `package/custom/` 目录
2. 清理原有的同名包 Makefile（避免冲突）

### 方法 3：使用 make menuconfig 手动选择

1. 运行 `make menuconfig`
2. 浏览菜单选择你需要的包
3. 保存配置
4. 导出最小配置：`./scripts/diffconfig.sh &gt; .config.new`

## 如何添加新设备的支持

### 第一步：了解设备基本信息

你需要知道：
- 设备的 SoC 型号（如 MediaTek MT7981, Qualcomm IPQ6018 等）
- CPU 架构（arm, arm64, x86_64 等）
- 设备的 Flash 和 RAM 大小
- 网络接口配置

### 第二步：添加设备到工作流

编辑 `.github/workflows/` 下的工作流文件，在 `matrix.include` 中添加你的设备：

```yaml
matrix:
  include:
    # 现有设备...
    - device: YOUR-DEVICE-NAME
      repo_url: 'https://github.com/m0eak/fanchmwrt.git'
      repo_branch: '你的分支名'  # 或现有分支
      config_file: './config/your-device.config'
      workflow_name: 'YOUR-DEVICE-NAME'
```

### 第三步：创建设备配置文件

在 `config/` 目录下创建 `your-device.config`：

```bash
# 1. 选择目标架构和设备
CONFIG_TARGET_&lt;target&gt;=y
CONFIG_TARGET_&lt;target&gt;_&lt;subtarget&gt;=y
CONFIG_TARGET_&lt;target&gt;_&lt;subtarget&gt;_DEVICE_&lt;device-profile&gt;=y

# 2. 添加你需要的插件（参考 mt5000.config）
CONFIG_PACKAGE_luci-app-xxx=y
# ...
```

### 第四步：添加 DTS（设备树）文件（如果需要）

如果 OpenWrt 源码中还没有你设备的 DTS 文件，你需要：

1. **获取或创建 DTS 文件**：
   - 参考同 SoC 的其他设备的 DTS 文件
   - 通常位于：`target/linux/&lt;target&gt;/dts/`

2. **在 scripts-part1.sh 中添加处理逻辑**：

编辑 [sh/scripts-part1.sh](file:///workspace/sh/scripts-part1.sh)，添加新的 `elif` 块：

```bash
# --- 逻辑块：处理 YOUR-DEVICE ---
elif [[ "$WORKFLOW_NAME" == "YOUR-DEVICE-NAME" ]]; then
    echo "&gt;&gt;&gt; 检测到设备: $WORKFLOW_NAME。开始执行特定修改"

    # 修改默认 IP
    sed -i 's/192.168.1.1/192.168.5.1/g' package/base-files/files/bin/config_generate
    echo "YOUR-DEVICE IP 修改为 192.168.5.1"

    # 如果需要替换 DTS 文件
    CUSTOM_DTS_URL="https://raw.githubusercontent.com/.../your-device.dts"
    CUSTOM_DTS_TARGET="target/linux/&lt;target&gt;/dts/your-device.dts"
    curl -fL "$CUSTOM_DTS_URL" -o "$CUSTOM_DTS_TARGET"

    # 其他特定修改...
fi
```

3. **DTS 文件要点**：
   - 配置正确的内存大小
   - 配置正确的 Flash 分区
   - 配置 LED 和按键
   - 配置网络接口（MAC 地址、PHY 等）

### 第五步：添加设备 Profile（如果 OpenWrt 源码中没有）

你需要：
1. 在 `target/linux/&lt;target&gt;/image/` 目录下添加设备 Makefile
2. 定义设备的编译参数、内核命令行等
3. 添加固件生成命令

参考同目录下其他设备的配置。

## 本地编译与 GitHub Actions 编译

### GitHub Actions 编译的优势
- ✅ 无需配置本地环境
- ✅ 自动处理依赖
- ✅ 支持定时自动编译
- ✅ 编译产物自动发布
- ✅ 可以缓存工具链加速

### 本地编译的优势
- ✅ 调试更方便
- ✅ 可以随时修改和测试
- ✅ 不受 GitHub Actions 时间限制
- ✅ 更快速的迭代

## 常见问题

**Q: 编译失败怎么办？**
A: 首先尝试使用单线程编译查看详细错误：`make -j1 V=s`

**Q: 如何减小固件体积？**
A: 在 .config 中去掉不需要的包，使用 `CONFIG_PACKAGE_xxx=n` 或去掉相关行

**Q: 如何添加自定义文件？**
A: 将文件放入 `files/` 目录，保持相应的目录结构，会自动复制到固件的 `/` 目录下

**Q: 如何修改默认设置（如主机名、密码等）？**
A: 查看 `default-settings-m0eak/` 目录，或在 files/etc/uci-defaults/ 下添加脚本
